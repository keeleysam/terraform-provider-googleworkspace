# googleworkspace_chrome_policy_file (Resource)

Uploads a file (e.g. a wallpaper image) to Google Chrome Policy and returns a public download URI that can be referenced in other Chrome policies.

This is a create-only resource backed by the [`media.upload`](https://developers.google.com/chrome/policy/reference/rest/v1/media/upload) API. Updates to the file or policy field force recreation. Deletion only removes the resource from Terraform state — the uploaded file cannot be deleted via the API.

Chrome Policy Schema resides under the `https://www.googleapis.com/auth/chrome.management.policy` client scope.

## Example Usage

```terraform
resource "googleworkspace_chrome_policy_file" "wallpaper" {
  policy_field = "chrome.users.Wallpaper.wallpaperImage"
  file_path    = "${path.module}/wallpaper.jpg"
}

resource "googleworkspace_chrome_policy" "wallpaper" {
  org_unit_id = googleworkspace_org_unit.example.id
  policies {
    schema_name = "chrome.users.Wallpaper"
    schema_values = {
      wallpaperImage = jsonencode(googleworkspace_chrome_policy_file.wallpaper.download_uri)
    }
  }
}
```

## Schema

### Required

- `policy_field` (String) The fully qualified policy schema and field name this file is uploaded for, e.g. `chrome.users.Wallpaper.wallpaperImage`. The API uses this to validate the content type of the file.
- `file_path` (String) Path to the local file to upload.

### Read-Only

- `id` (String) The ID of this resource.
- `file_sha256` (String) SHA-256 hash of the uploaded file. Changes to the local file will produce a new hash and trigger resource recreation.
- `download_uri` (String) The public download URI for the uploaded file, returned by the API. Use this value to reference the file in Chrome policy configurations.

## Import

Existing uploaded files can be imported using a pipe-separated ID containing the policy field, the local file path, and the download URI:

```shell
terraform import googleworkspace_chrome_policy_file.wallpaper 'chrome.users.Wallpaper.wallpaperImage|./wallpaper.jpg|https://storage.googleapis.com/chromeos-mgmt/...'
```

The local file must exist at the given path so that its SHA-256 hash can be computed. After import, `terraform plan` will show no changes provided the local file matches the one that was originally uploaded.
